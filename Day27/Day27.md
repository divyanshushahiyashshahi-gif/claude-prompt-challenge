# Day 27 – Prior Authorization Story Simulator

## 📌 Challenge

Built an interactive healthcare education simulator that explains the **Prior Authorization (PA)** workflow through a story-based experience.

The application is designed to make a complex healthcare insurance process easier to understand using conversations, educational explanations, and user choices.

---

## 🎯 Objective

Create a beginner-friendly simulator that demonstrates how a Prior Authorization request moves from the healthcare provider to the insurance payer and how appeals and approvals work.

---

## 🛠️ Tech Stack

- HTML5
- Tailwind CSS (CDN)
- Vanilla JavaScript
- Single File Architecture

---

## ✨ Features

- Interactive story with 8 scenes
- Rahul (Patient) and Priya (Healthcare Operations Specialist) chat interface
- Narrator and Doctor messages
- Animated progress bar
- Scene progression
- Two choices after every scene
- Append-only chat feed
- Responsive UI
- Restart Story option
- Typing animations
- Modern healthcare design

---

## 📚 Story Flow

1. Doctor Visit
2. Insurance Roadblock
3. What is Prior Authorization?
4. Insurance Review
5. Denial
6. Appeal
7. Approval
8. Final Takeaways

---

## 🧠 Key Learnings

- Prior Authorization is not simply paperwork—it helps insurers verify that treatments meet medical and coverage requirements.
- Clinical documentation plays an important role in approval decisions.
- Step therapy may be required before certain medications are approved.
- A denial is not necessarily permanent and can often be appealed with additional documentation.
- Effective documentation and communication improve approval outcomes.
